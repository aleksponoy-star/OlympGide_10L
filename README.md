ОлимпГид

Веб-платформа для подбора олимпиад и вузов на основе актуальных требований и льгот.

Цель проекта

Создать удобный сервис, который собирает информацию об олимпиадах, вузах, факультетах и льготах в одном месте и помогает школьникам быстрее ориентироваться в вариантах поступления.

Актуальность

Информация о льготах, уровнях олимпиад и правилах учёта достижений разбросана по десяткам сайтов. Из-за этого учащиеся тратят много времени на поиск и могут пропускать сроки. Проект решает эту проблему, объединяя данные в единую систему с удобным поиском и фильтрами.

Задачи проекта

собрать и структурировать требования к системе;

разработать пользовательские сценарии;

выбрать технологический стек;

спроектировать архитектуру сайта;

подготовить и привести к единому формату данные об олимпиадах и вузах;

разработать интерфейс;

реализовать поиск, фильтры, личный кабинет и систему уведомлений;

обеспечить корректную работу на ПК и мобильных устройствах.

Основные пользовательские сценарии

1. Подбор олимпиад для конкретного вуза и факультета

Пользователь выбирает вуз → факультет → получает список олимпиад, которые дают льготы на выбранное направление.

2. Подбор вуза по направлению
   
Пользователь выбирает направление (например, «математика») → получает список подходящих вузов → выбирает конкретный → получает олимпиады, дающие льготы для поступления.

3. Сценарий администратора
 
Администратор загружает и обновляет данные:

олимпиады,
вузы,
направления,
факультеты,
льготы и сроки.
Структура базы данных
Пользователи: id, логин, пароль

Избранные олимпиады: пользователь_id, олимпиада_id, дата_добавления

Олимпиады: id, название, предмет, льготы, уровень, сроки, этапы

Вузы: id, название, город, принимаемые льготы

Факультеты: id, название, направление, вуз_id, льготы

Связи:

факультет → вуз

направление → вуз

олимпиада → льготы → факультет

пользователь → избранные олимпиады → олимпиады

Используемые технологии

HTML, CSS, JavaScript, Python (Flask/Django), SQLite или Firebase.

Архитектура

клиентская часть: интерфейс, формы поиска, отображение результатов;

серверная часть: работа с данными, фильтрация, авторизация;

база данных: хранение олимпиад, вузов, льгот, пользователей.

Функции, планируемые к реализации

поиск олимпиад по вузу, направлению, предмету;

фильтры по уровням, срокам, льготам;

личный кабинет;

избранные олимпиады;

напоминания о сроках;

автоматическое обновление данных.
